SplitX — Smart Group Expense Tracker

Designed and developed during a 36-hour hackathon

Full-stack · React Native · Node.js · MongoDB

View Code

Overview

SplitX is a group expense tracking application designed to simplify how shared expenses are recorded, split, and managed. It enables users to create groups, add members, and track shared financial activity through a structured backend system.

Problem

In scenarios like trips, shared living, or events, expense tracking is often inefficient due to manual calculations, lack of visibility, and absence of a centralized system.

Solution

Designed a system that centralizes expense tracking using structured data models and REST APIs, ensuring accuracy, scalability, and ease of use.

Tech Stack

Frontend: React Native  ·  Backend: Node.js, Express.js  ·  Database: MongoDB (Mongoose)

System Design

The system follows a RESTful architecture with clear separation between API layer, business logic, and data layer.

  • Node.js + Express handles API routing
  • MongoDB manages data using schema-based modeling
  • Mongoose enables structured relationships
  • Endpoints ensure modular and scalable design

API Design

  • POST /users → Create user
  • POST /groups → Create group
  • GET /groups → Fetch all groups
  • POST /expenses → Add expense
  • GET /groups/:id/expenses → Get group expenses

Database Design

  • User: name, email
  • Group: name, members (User IDs)
  • Expense: groupId, amount, paidBy, splitBetween

Uses ObjectId references to maintain relationships and ensure scalability.

Key Features

  • Group creation and management
  • Expense tracking and splitting
  • RESTful API structure
  • Centralized error handling
  • Data validation and constraints

Highlight: Cascade Deletion

Implemented cascade deletion to automatically remove all related expenses when a group is deleted, ensuring database consistency and preventing orphan data.
