Savor — Decision-First Food Ordering

A UX + frontend project focused on reducing decision fatigue in food ordering through clarity and structured choices.

UI/UX · Frontend · Product Thinking

Live Demo

Problem

Most food delivery apps prioritize engagement and scrolling instead of decision-making. Users often feel overwhelmed due to too many options, cluttered UI, and weak personalization.

This leads to decision fatigue and slower ordering experiences.

Solution

Savor introduces a decision-first approach — guiding users toward the right choice instead of showing endless options.

The focus is on clarity, structure, and confidence in decision-making.

Key Features

  • Budget-first discovery using sliders and presets
  • Curated results with best match and limited options
  • Dish clarity with rating, delivery time, and pricing
  • Decision support like “Decide for me”
  • Clean UI focused on minimal and distraction-free experience

Research Insights

  • Too many options cause decision fatigue
  • Cluttered UI increases cognitive load
  • Poor hierarchy slows down user decisions

Tech Stack

  • React + Vite for frontend
  • Tailwind CSS for styling
  • React Router for navigation
  • Context API for state management
  • Netlify for deployment

Impact

Savor reduces time-to-decision and improves user confidence by focusing on clarity instead of quantity.

Future Scope

  • Backend integration with real APIs
  • Payment system integration
  • Personalized recommendations
